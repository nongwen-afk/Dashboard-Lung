# UI/UX Guidelines

## Design Philosophy

Do not make the dispatcher think. Make the dispatcher confirm.

## Shared Design System

Admin and Dispatcher views should share:

- Typography
- Buttons
- Cards
- Tables
- Status badges
- Icons

But they should differ in complexity.

## Admin UI

Admin is desktop-first.

Admin sees:

- Dashboard
- Routes
- Vehicles
- Drivers
- Reserve Drivers
- Analytics
- Settings

Admin can manage data.

## Dispatcher UI

Dispatcher is mobile-first.

Dispatcher sees:

- Route selector
- Situation card
- Recommendation card
- Confirm button
- Operations table
- Event timeline

Dispatcher should not see complex menus.

## Accessibility

- Large text
- Large buttons
- Icon + text status
- Avoid using color alone
- Keep actions obvious

## Status Display

Use icon + text.

Examples:

- ✅ Active
- 🏥 Leave
- ❌ Absent
- 🚗 Breakdown
- 🔄 Reserve Replacement
- ⚠️ Pending Approval
